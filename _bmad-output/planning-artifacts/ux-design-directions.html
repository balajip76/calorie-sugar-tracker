Direction A: Centered Hero

The daily total dominates the screen as a single large centred number. Tapping it opens the entry sheet. History scrolls below. Insights at the bottom. The most focused, single-element layout.

Direction B: Split Cards

Calories and sugar get equal visual weight as two side-by-side cards. An explicit full-width "Add Entry" button below. Both metrics are first-class citizens from the very first glance.

Direction C: Minimal + FAB

Maximum whitespace and breathing room. Large hero number, line-separated entries instead of cards, and a floating action button (FAB) for entry. The most restrained layout.

Direction D: Compact Dashboard

Everything above the fold: stat cards, inline insights with pill selector, and a compact entry list. Most information-dense while still clean.

Direction E: Entry Sheet (Open State)

Shows the bottom sheet entry flow: smart search field, calorie and sugar inputs, and a full-width Log button. This is the core interaction moment.

Direction F: Zero State (First Open)

What the brand-new user sees: zeroes, a clear add button, empty insights. The interface must be self-evident without instruction.
